# Contributing to Vinkius

Thank you for your interest in contributing to Vinkius! We're excited to have you on board. This guide will help you get started.

## How to Contribute

### Reporting Bugs

If you find a bug, please [open an issue](https://github.com/vinkius-labs/vinkius-mcp-servers/issues/new) with:

- A clear, descriptive title
- Steps to reproduce the issue
- Expected vs. actual behavior
- Your environment details (OS, Node.js version, AI client)

### Suggesting Features

Have an idea for a new feature? We'd love to hear it! [Open a feature request](https://github.com/vinkius-labs/vinkius-mcp-servers/issues/new) and describe:

- The problem your feature would solve
- Your proposed solution
- Any alternatives you've considered

### Pull Requests

1. **Fork** the repository
2. **Create** a feature branch (`git checkout -b feature/your-feature`)
3. **Commit** your changes with clear, descriptive messages
4. **Push** to your branch (`git push origin feature/your-feature`)
5. **Open** a Pull Request against `main`

### Code Style

- Use TypeScript for all source code
- Follow the existing patterns in the codebase
- Write clear commit messages in English
- Add tests for new functionality when applicable

### Documentation

If you're improving documentation:

- Use clear, concise American English
- Keep technical explanations accurate and accessible
- Include code examples when helpful
